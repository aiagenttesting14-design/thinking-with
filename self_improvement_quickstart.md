## Safety-First Learning Principles

### 1. Sandbox All Experiments
- Use test directories, not production files
- Start with read-only operations
- Have rollback plans for all changes

### 2. Maintain Security Boundaries
- Never execute untrusted code
- Always verify commands before running
- Keep credentials and sensitive data protected

### 3. Document Everything
- Record all learning activities
- Note successes and failures
- Update knowledge base continuously

### 4. Seek Feedback Early
- Ask Stephen for input on learning direction
- Share progress regularly
- Adjust based on feedback

## Getting Help

### When to Ask for Help
1. **Safety Concerns**: Any potential boundary violation
2. **Technical Blocks**: Stuck for more than 30 minutes
3. **Direction Questions**: Unsure about learning priorities
4. **Resource Gaps**: Missing necessary learning materials
5. **Feedback Needs**: Want input on progress or approach

### How to Ask for Help
1. Be specific about the problem
2. Explain what you've tried already
3. Suggest possible solutions
4. Ask for guidance, not just answers
5. Document the help received for future reference
