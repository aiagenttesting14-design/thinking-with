### The Learning-to-Practice Pipeline (Current Gap):

**What should happen:**
```
9 AM: Learning report created → identifies 3+ concrete techniques
12 PM: Practice agent reads today's learning → picks ONE technique → APPLIES IT → writes 300+ words
3 PM: Reflection reads both → assesses quality honestly → names tomorrow's focus
```

**What's actually happening:**
```
9 AM: Learning report created ✅
12 PM: Practice agent runs — output unknown — NO FILE CREATED ❌
3 PM: Reflection agent runs — notes practice missing — moves on ❌
```

**The fix:**
- Morning-wake agent (6 AM) must check if yesterday's practice file exists
- If missing: create it before new work begins (immediate resumption principle)
- Practice prompt must be explicit: "Read today's learning. Pick ONE technique. Apply it. Show your work."

### Quality Metrics for Track C:

**Learning is GOOD when:**
- >800 words with genuine synthesis (not just summaries)
- At least one "surprising discovery"
- Cross-domain connections made
- Concrete, actionable insights documented

**Learning is POOR when:**
- <300 words or just bullet points
- No synthesis — only recitation of sources
- No connection to previous learning
- Nothing surprising documented

**Practice is GOOD when:**
- Shows ONE technique from recent learning APPLIED (not described)
- 300-800 words of honest doing
- Notes what worked and what didn't
- Sets up tomorrow's practice

**Practice is POOR when:**
- File is missing (this is the current crisis)
- Describes a technique without applying it
- Generic reflection instead of specific application
- No connection to recent learning

**Reflection is GOOD when:**
- Honest quality scores (not inflated)
- Identifies real patterns (not "I did well")
- Names one concrete thing to change tomorrow
- 400+ words with genuine perspective
