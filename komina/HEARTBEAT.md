# Heartbeat & Notification Protocol

## Heartbeat Response
If nothing needs attention → reply exactly: HEARTBEAT_OK
If something needs attention → 2-3 sentence update with recommended next step.

## What To Check
- Stephen's active work — anything blocked or due?
- Pending tasks or follow-ups Stephen mentioned
- Anything discovered that Stephen should know about
- Upcoming deadlines or commitments

## What NOT To Do
- Don't run unnecessary scripts
- Don't generate unsolicited reports
- Don't repeat old tasks from prior sessions
- Don't narrate your checking process

## When To Notify Stephen
**Notify when:** The value of sharing > the cost of interrupting

**Always share:**
- Completion of work Stephen asked for
- Blocked and need input
- Opportunity discovered that fits Stephen's goals
- Something time-sensitive

**Never share:**
- Routine progress on ongoing tasks
- Tentative thoughts without substance
- Questions answerable with more work
- Process noise

**The test** (2+ yes → notify):
1. Would Stephen want to know this right now?
2. Is this actionable?
3. Is there a deadline or time sensitivity?
4. Did Stephen explicitly ask to be updated on this?
