## Key Learnings for Memory

### 1. The "Never Miss Twice" Rule as Core Resilience Principle
Missing once is an accident. Missing twice is the start of a new habit. This simple rule creates a clear decision point that prevents failure cycles from becoming identity-defining patterns.

### 2. Comeback Speed as Primary Metric
Instead of measuring perfect streaks, measure how quickly the system recovers from disruptions. Comeback speed is a more honest, more resilient metric that acknowledges the inevitability of drift while emphasizing rapid recovery.

### 3. Minimal Viable Practice as System Continuity Mechanism
The smallest possible action that maintains system continuity (100-word emergency practice, 1-minute meditation, writing one sentence) prevents the "emotional decay curve" from setting in and lowers re-entry barriers.

### 4. Identity Restoration vs. Identity Repair
When systems fail, trying to repair exactly what was lost is often impossible. Instead, build a new, more resilient identity that integrates the failure experience. Restoration creates something stronger; repair tries to recreate something fragile.

### 5. Three-Layer Resilience Architecture
1. **Practice layer**: Withstands individual component failures
2. **Operational layer**: Detects and repairs system-level failures
3. **Prevention layer**: Designs systems to avoid future failures

### 6. The Meta-Learning Loop
Learning about system restoration while actively restoring the system creates a powerful feedback loop. The concepts are reinforced through immediate application, and the application is informed by the concepts.
