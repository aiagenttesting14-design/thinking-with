## What Was Learned

The focus file named the right problem: **designing a verification-first operating loop for autonomous agents.**

The two autonomous learn files sharpened that into specific operating lessons:

1. **Multi-agent systems should stay boring.** One orchestrator, isolated workers, strong handoffs, minimal overlap.
2. **Browser reliability comes more from protocol than cleverness.** Snapshot discipline, explicit tab targeting, re-snapshotting after UI changes, and separate verification are the real stability gains.
3. **Verification has to survive ambiguity.** Files, browser actions, cron surfaces, and external state can disagree; the loop needs trusted sources and mismatch handling.
4. **Return is architectural as well as existential.** The same logic that keeps a system honest across resets also keeps identity honest across session gaps.
